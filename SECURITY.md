# Security Policy

## Supported Versions

Security updates are provided for the latest code on the `main` branch.

| Version | Supported |
| ------- | --------- |
| main    | Yes       |
| older tags/branches | No |

## Reporting a Vulnerability

Please report security vulnerabilities privately.

1. Open a private report using GitHub Security Advisories:
   - https://github.com/AdvaitPalve11/Issue-Tracking-System/security/advisories/new
2. Include clear reproduction details:
   - Affected endpoint or component
   - Steps to reproduce
   - Expected impact
   - Any proof-of-concept details
3. If possible, include versions and environment details.

Do not open public issues for security vulnerabilities.

## Response and Disclosure Process

- Initial triage response target: within 3 business days
- Status update target: within 7 business days
- Fix timeline depends on severity and complexity

When a report is validated, we will:

1. Confirm impact and severity
2. Prepare and test a fix
3. Coordinate a responsible disclosure timeline
4. Publish advisory details after a fix is available

## Scope

This policy covers:

- Backend API and authentication behavior
- Frontend application code in `frontend/`
- Repository configuration and deployment files
- Third-party dependency risk in this repository

## Safe Harbor

If you act in good faith, avoid privacy violations or service disruption, and do not publicly disclose before a fix is available, we will treat your research as responsible disclosure.
